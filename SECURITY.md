# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

This is an educational demonstration project and is **not intended for clinical use**. 
However, if you discover a security vulnerability that could affect derivative works:

1. **Do not** open a public GitHub issue
2. Email: maximilian.dressler@stud.uni-heidelberg.de
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact assessment

Expected response time: 7 business days

## Scope

Security concerns relevant to this project include:
- Dependency vulnerabilities in `requirements.txt`
- Data leakage in example outputs
- Injection vulnerabilities in data parsing
